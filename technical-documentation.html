01 — Overview

System Overview

The Executive Management System is a multi-portal web application designed for executive-level operations. It provides three specialized portals — Travel, Task, and IT Management — all controlled by a central Admin panel, accessible through a single unified login.

05 — Data Flow

Authentication & Data Flow

The system uses a custom session management and hashed password system built on top of browser-native APIs.

Login Flow
1

User Enters Credentials

Username and password entered on the login page. Password is immediately processed by hashPass() using SubtleCrypto SHA-256.

2

Hash Comparison

The hashed input is compared against stored SHA-256 hashes in localStorage['users'] and localStorage['users_executive_it'].

3

Session Creation

On match, a session object is stored: { loggedIn: true, username, name, role, isAdmin } in localStorage['session'].

4

Portal Redirect

User is automatically redirected to their designated portal based on session.role — no manual selection required.

5

Portal Auth Guard

Every portal runs initAuth() on load — checks session validity and cross-redirects unauthorized roles instantly.

Data Storage Pattern
A

User-Scoped Storage

Data is stored per-user using namespaced keys: trips_{username}, tasks_{username}, it_maintenance_{username}

B

Admin Cross-User Access

Admin users can enumerate all localStorage keys matching a prefix to aggregate data across all users in real-time.

C

ID Generation

Each record receives a unique ID via Date.now() + Math.random() to prevent collisions during rapid creation.

D

JSON Serialization

All data stored as JSON strings via JSON.stringify() and retrieved with JSON.parse() with null-safe defaults ('[]').

E

Cross-Module Sync

Technical Support daily jobs automatically create a linked Support Ticket entry using accomplishmentId for traceability.

Password Hashing Code Pattern
// SHA-256 hashing using Web Crypto API (SubtleCrypto)
function hashPass(pass) {
  var enc = new TextEncoder();
  return crypto.subtle.digest('SHA-256', enc.encode(pass))
    .then(function(buf) {
      return Array.from(new Uint8Array(buf))
        .map(b => b.toString(16).padStart(2, '0')).join('');
    });
}
06 — Security

Security Model

The system implements multiple security layers despite being entirely client-side — from hashed passwords to role isolation and data scoping.

enhanced_encryption

SHA-256 Password Hashing

All passwords are hashed using the browser's native SubtleCrypto.digest('SHA-256') API before storage. Passwords are never stored in plaintext. The hash is computed asynchronously and the original password is not retained in memory longer than needed.

alt_route

Role-Based Access Control (RBAC)

Every portal runs initAuth() on initialization. If the session role does not match the portal, the user is immediately redirected to their correct portal. Cross-portal access is blocked at the routing level.

storage

User-Scoped Data Isolation

All stored data (trips, tasks, logs, IT records) is namespaced by username: trips_{username}. Regular users can only read/write their own data. Only admin-flagged accounts can enumerate and aggregate across all user namespaces.

shield

IT Portal Account Isolation

Executive IT user accounts are stored in a separate key (users_executive_it) isolated from the main user store, providing an additional layer of separation for IT staff.

verified_user

Session Validation on Every Load

Each portal page validates the session object on startup. Missing, malformed, or role-mismatched sessions result in immediate redirect to the login page — no unauthenticated access is possible.

security

HTML Output Escaping

All dynamically rendered user content passes through escapeHtml() which sanitizes &, <, >, and " to prevent XSS injection attacks.

08 — User Roles

User Role Matrix

The system supports 4 distinct roles, each with access to a specific portal and scoped data visibility. Roles are assigned by the Administrator during account creation.

Role Portal Access Data Scope Admin Flag Key Capabilities
administrator Admin Control Panel All users' data, all modules Always User CRUD, data export/import, global search, system info
executive_path Executive Path (Travel) Own trips only (or all if portal admin) Optional Trip planning, travel calendar, itinerary, reports
executive_task Executive Task (Productivity) Own tasks/plans/logs only (or all if portal admin) Optional Task management, planner, daily log
executive_it Executive IT Own IT data only (or all if portal admin) Optional IT services, maintenance, inventory, tickets, system monitor
info
Portal Admin Flag
Any user of any role can be flagged as a Portal Admin by the System Administrator. Portal admins can view all users' data within their portal (not just their own), making them supervisors within their domain without granting full system admin access.
